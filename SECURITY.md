# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this package, please report it privately via [GitHub Private vulnerability reporting ↗](https://github.com/teqbench/tbx-mat-dialogs/security/advisories/new) rather than opening a public issue or pull request.

This channel keeps the report confidential while it is being assessed. The maintainers will acknowledge receipt and coordinate a fix and disclosure timeline with the reporter.

## Supported Versions

Only the latest published major version receives security patches. Older majors are end-of-life on release of the next major.

## Scope

In scope:

- Code published to [GitHub Packages ↗](https://github.com/orgs/teqbench/packages) under `@teqbench/tbx-mat-dialogs`.
- The build configuration, [CI/CD workflows ↗](https://docs.github.com/en/actions), and release process in this repository.

Out of scope:

- Vulnerabilities in upstream dependencies — report those to the upstream maintainers. If a CVE in an upstream dependency affects this package, the maintainers will track the dependency upgrade in a separate issue.
- Issues that depend on consumer-side misconfiguration outside this package's control.

## Acknowledgement

Reports made through the private channel and confirmed as valid will be acknowledged in the release notes for the patched version unless the reporter requests otherwise.
